# CA0063
規則セット ファイルの読み込みに失敗したか、依存規則セット ファイルのいずれかの読み込みに失敗しました。  
  
 規則セット ファイルの読み込みに失敗したか、依存規則セット ファイルのいずれかの読み込みに失敗しました。  
  
 指定された規則セットが見つからなかったか、規則セットに含まれている規則セット ファイルのいずれかが見つかりませんでした。  規則セットに含まれているすべての規則セットがディスク上に存在すること、およびプロジェクトで [!INCLUDE[vstecmsbuild](../build/includes/vstecmsbuild_md.md)] の **CodeAnalysisRuleSetDirectories** プロパティによって適切な規則セット ディレクトリが指定されていることを確認してください。  
  
 エラーをデバッグするには、規則セット ファイルをテキスト エディターで確認します。  規則セット ファイルのパスを見つけるには、ソリューション エクスプローラーでプロジェクトを右クリックし、**\[プロパティ\]** をクリックして、**\[コード分析\]** をクリックします。  **\[この規則セットを実行\]** で規則セット ファイルが選択されていることを確認します。  規則セットのパスが説明フィールドに表示されます。  
  
 すべての **Include** 要素の **Path** 属性値を確認します。  インクルード パスでは、親または現在の規則セット ファイルの相対パス、環境変数、および絶対パスを使用できます。  たとえば、次のようになります。  
  
```  
<Include Path="%PUBLIC%\Documents\RuleSets\alldesign.ruleset" Action="Default" />  
<Include Path="..\alldesign.ruleset" Action="Default" />  
<Include Path="C:\Rulesets\alldesign.ruleset" Action="Default" />  
```  
  
 これらの各インクルード パスを調べ、すべて有効であることを確認します。  
  
 場合によっては、規則セットの場所が [!INCLUDE[vstecmsbuild](../build/includes/vstecmsbuild_md.md)] プロパティに依存していることがあります。  [!INCLUDE[vstecmsbuild](../build/includes/vstecmsbuild_md.md)] の プロパティは規則セットから参照できません。  この問題を回避するには、[!INCLUDE[vstecmsbuild](../build/includes/vstecmsbuild_md.md)]の **CodeAnalysisRuleSetDirectories** のプロパティで追加の検索パスを指定します。  この場合、**Include** 要素の **Path** 属性では規則セットの名前のみ指定します。
